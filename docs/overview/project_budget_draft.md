## 작성 전제

- 본 문서는 승인 이전 단계의 예산 구조 초안이다.
- 현재는 조직별 단가, 투입률, 인프라 구성, 라이선스 정책이 확정되지 않았으므로 금액 자체보다 예산 항목과 산정 기준을 우선 정의한다.
- 인력 투입은 실제 진행에 따라 변경될 수 있으므로, 예산은 고정 인원 수가 아니라 역할별 투입 범위와 기간 조합으로 재산정해야 한다.
- 현재 승인 기준 일정은 `Phase 1 2026년 8월`, `Phase 2 프로토타입 2026년 10월`이며, 본 문서는 그 일정을 전제로 한 범위형 예산 검토 초안이다.

## 1. 예산 항목 구조

| 구분 | 세부 항목 | 산정 기준 초안 | 비고 |
| --- | --- | --- | --- |
| 인건비 | PM/기획 | 참여 기간 x 역할별 투입률 | 초기 분석/조정 비중 높음 |
| 인건비 | 아키텍트 | 설계 구간 중심 투입 | 표준 모델/연계 구조 정의 |
| 인건비 | 개발 | 구현 범위 x 병렬 개발 수요 | 연계 시스템 수에 따라 증감 |
| 인건비 | 운영/보안 | 정책 검토 및 시범 운영 지원 | 인증/권한/감사 검토 포함 |
| 인건비 | 데이터/운영 기준정보 관리 | 조직/인력 기준정보, 마이그레이션 검토, 운영 정책 정비 | 조직 개편 대응 포함 |
| 인프라 | 개발/검증 환경 | 서버, DB, 스토리지, 네트워크 | 온프레미스/클라우드에 따라 상이 |
| 소프트웨어 | 연계/모니터링/분석 도구 | 신규 도입 여부 기준 | 기존 보유 라이선스 확인 필요 |
| 검증 | 성능/보안/품질 검토 | 필요 범위에 따라 산정 | 내부/외부 검토 여부 확인 |
| 운영 준비 | 교육, 전환, 문서화 | 시범 범위와 사용자 수 기준 | Phase 1 말기 반영 |
| 예비비 | 범위 변경/예상 외 연계 이슈 | 총예산의 일정 비율 | 초기에는 별도 버퍼 권장 |

## 2. 인력 비용 산정 원칙

- 인력은 `고정 N명 x 전체 기간` 방식보다 `역할별 투입률 x 단계별 기간` 방식이 적절하다.
- 분석/설계 단계와 구현/검증 단계의 필요 역량 구성이 다르므로 월별 평균 인원으로 단순화하지 않는다.
- 연계 대상 시스템이 늘어나면 개발과 검증 비용이 비선형적으로 증가할 수 있다.
- 시범 적용 범위가 커질수록 운영 지원과 사용자 검증 비용이 증가한다.

## 3. 예산 시나리오 제안

### 시나리오 A. 최소 범위 시범

- 우선 연계 대상 2~3개
- 핵심 조회와 운영 기능 중심
- `AI` 보조 기능은 설계 검토 수준

적합 상황:

- 빠른 `PoC` 또는 `MVP` 검증이 목적일 때

### 시나리오 B. 표준 Phase 1

- 우선 연계 대상 4~6개
- 조회, 추적성, 운영 기능, 일부 품질 지표 포함
- 조직/인력 기준정보, 조직 변경 마이그레이션, 공유 캘린더/부재 관리 최소 기능 포함
- `AI` 보조 기능은 제한적 파일럿 포함

적합 상황:

- 실제 시범 운영과 투자 판단을 함께 진행할 때

일정 메모:

- `Phase 1 2026년 8월`, `Phase 2 프로토타입 2026년 10월` 목표라면 현실적으로 가장 기준이 되는 시나리오다.
- 다만 이 경우에도 우선 연계 대상의 깊이를 조절해야 하며, 일부 기능은 조회 중심 `MVP`로 제한하는 것이 바람직하다.
- 현재 상위 기안과 실행 계획은 본 시나리오를 승인 기준선으로 사용한다.

### 시나리오 C. 확장형 Phase 1

- 복수 조직/복수 프로젝트 대상
- 연계 대상 다수, 운영 정책 정교화 포함
- `AI` 보조 기능과 운영 도구까지 동시 추진

적합 상황:

- 중장기 플랫폼화 전제를 갖고 초기부터 확장성을 확보하려 할 때

## 4. 예산 확정 전 확인 필요 사항

- 역할별 내부 인건비 기준 또는 외주 단가
- 온프레미스/클라우드 배포 모델
- 신규 라이선스 구매 필요 여부
- 보안성 검토 및 성능 검증 범위
- 시범 조직과 대상 사용자 수
- 조직 개편 대응 빈도와 조직/인력 마이그레이션 운영 인력 기준
- 공유 캘린더 및 부재 관리의 내부 개발 여부 또는 외부 연계 여부

## 5. 예산 산정시 주의사항

- 인력 투입은 실제 진행 상황에 따라 늘거나 줄 수 있으므로, 확정 예산은 단계별 게이트를 기준으로 재승인하는 방식이 바람직하다.
- `AI` 기능은 기반 플랫폼 안정화 이후 적용하는 것이 총비용과 리스크를 낮출 수 있다.
- 외부 시스템 API 제약이나 데이터 정합성 문제는 예상보다 큰 추가 비용을 유발할 수 있으므로 예비비를 별도 반영하는 편이 안전하다.
- 2026년 8월과 2026년 10월 목표를 유지하려면 일정 지연을 인력 추가로만 해결하기보다 범위와 우선순위 조정을 병행해야 한다.

## 6. 다음 단계 제안

- 실제 단가표를 확보한 뒤 시나리오별 예산표를 별도 작성
- `최소/기준/확장` 시나리오별 투자 범위와 승인 게이트 정의
- 단계별 승인 예산과 집행 기준 정의
- 투자 대비 효과를 정량 지표와 연결해 보완
