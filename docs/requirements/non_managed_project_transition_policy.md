- 문서 목적: 관리 외 프로젝트를 관리 대상 프로젝트로 전환하는 기준과 절차를 정의한다.
- 범위: 전환 트리거, 승인 기준, 필수 보완 항목, 상태 전이, 운영 통제

## 1. 정책 개요

관리 외 프로젝트는 최소 정보 등록 상태로 유지되지만, 진행 중 중요도가 높아지거나 연계 필요성이 커지면 관리 대상 프로젝트로 전환될 수 있어야 한다. 이 문서는 그 전환의 기준과 절차를 정의한다.

## 2. 전환 필요 상황 초안

- 외부 연계 범위가 확대되는 경우
- 품질, 테스트, 배포 이력의 정식 관리가 필요한 경우
- 경영 보고 또는 감사 대상에 포함된 경우
- 프로젝트 규모나 중요도가 증가한 경우
- 다수 시스템 간 추적성 확보가 필요한 경우

## 3. 전환 요청 절차

1. 전환 요청 생성
2. 전환 사유 입력
3. 필수 정보 보완
4. 승인 검토
5. 관리 대상 프로젝트로 전환
6. 연계 범위 확장 및 운영 기준 적용

## 4. 승인 판단 기준 초안

- 관리 범위 포함 필요성이 명확한가
- 필수 메타데이터가 충족되었는가
- 보안, 권한, 운영 기준 적용이 가능한가
- 우선 연계 대상 시스템 연결 필요성이 확인되었는가
- 보고/감사 또는 운영 통제 측면에서 정식 관리가 필요한가

## 5. 전환 시 필수 보완 항목

- 과제 또는 관리 대상 식별자
- 담당 조직과 책임자
- 일정 정보
- 관련 시스템 목록
- 연계 우선순위
- 권한 및 보안 등급
- 문서 및 보고 필요 수준

## 6. 상태 전이 초안

| 현재 상태 | 이벤트 | 다음 상태 |
| --- | --- | --- |
| `light_registered` | 전환 요청 | `transition_requested` |
| `transition_requested` | 정보 보완 요청 | `supplement_required` |
| `supplement_required` | 보완 완료 | `under_transition_review` |
| `transition_requested` | 즉시 검토 착수 | `under_transition_review` |
| `under_transition_review` | 승인 | `converted` |
| `under_transition_review` | 반려 | `light_registered` |

## 7. 전환 후 처리 항목

- 관리 대상 프로젝트 식별자 부여
- 일반 과제 등록 기준 충족 여부 재검증
- 우선 연계 시스템 연결 계획 생성
- 권한과 운영 정책 재적용
- 필요한 경우 기존 간소 등록 정보와 새 관리 정보 병합

## 8. 운영 통제 원칙

- 전환 전후 데이터 연속성이 유지되어야 한다.
- 전환 이력과 승인 이력은 감사 가능해야 한다.
- 전환 이후에는 일반 과제와 동일한 관리 기준을 적용해야 한다.
- 전환 실패 또는 반려 시에도 기존 간소 등록 정보는 유지되어야 한다.

## 9. 오픈 이슈

- 전환 승인권자를 누구로 둘 것인가
- 전환 요청 시 자동으로 생성할 필수 체크리스트 범위를 어디까지 둘 것인가
- 전환 후 즉시 연계할 시스템과 후속 단계에서 연계할 시스템을 어떻게 나눌 것인가
