## 작업 목록

### 1. N+1 Query in Project Write Sync 성능 최적화
- **상태**: `done`
- **목표**: `backend/src/services/project_write.rs` 내의 `apply_for_run` 함수에서 발생하는 N+1 쿼리 성능 문제를 해결.
- **내용**:
  - `project_write.rs`에서 반복문 내부에서 `resolve_organization_id` 및 `resolve_workforce_id`를 호출하여 매 레코드마다 쿼리가 발생하는 문제를 확인.
  - 최적화를 위해 전체 페이로드를 스캔하여 `organization_code`와 `employee_number`를 추출하고, 이를 `HashSet`에 담아 중복을 제거.
  - PostgreSQL의 `ANY($1)`를 이용한 단일 배치 쿼리를 수행하여 ID 맵핑 정보를 `HashMap`에 저장.
  - 반복문 내부에서는 더 이상 쿼리를 수행하지 않고 `HashMap`에서 값을 조회하여 할당.
  - 기존 `resolve_*` 비공개 헬퍼 함수들을 제거.
  - `tests/bench_project_write.rs`를 추가하여 성능 개선 전후의 지표를 측정.
  - `bootstrap.rs` 테스트 코드의 단언(assertion)에서 발생하던 알파벳순 정렬 문제 수정.
- **결과**:
  - `apply_for_run` 성능이 대폭 향상됨 (5000개 레코드 기준 약 8.5초 소요에서 10ms 이하로 단축).
  - 전체 단위 테스트 및 성능 테스트 성공 확인.
