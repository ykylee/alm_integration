# 워크플로우 자동화 계획

- 문서 목적: 운영 문서와 작업 워크플로우를 보조하는 자동화 후보를 우선순위별로 정리하고, 실제 구현으로 내릴 수 있는 준비 상태를 만든다.
- 범위: 자동화 대상 선정, 우선순위, 입력/출력, 기대 효과, 구현 준비 상태, 후속 작업
- 대상 독자: 개발자, 운영자, AI agent, 문서 작성자
- 상태: draft
- 최종 수정일: 2026-04-17
- 관련 문서: `docs/operations/workflow_development_plan.md`, `docs/operations/workflow_task_execution.md`, `docs/operations/document_sync_policy.md`, `docs/operations/code_index_strategy.md`

## 1. 문서 개요

이 문서는 자동화 구현 자체가 아니라 “무엇부터 자동화할지”를 정리한 계획 문서다. 원칙은 문서를 대체하는 자동화보다, 누락을 줄이고 일관성을 확인하는 자동화부터 도입하는 것이다.

## 2. 자동화 우선순위

### 2.1 우선순위 1

- 백로그 템플릿 생성 보조
- 문서 메타데이터 누락 검사
- 허브/인덱스 링크 누락 검사
- 변경 경로 기반 영향 문서 후보 출력

### 2.2 우선순위 2

- 세션 종료 체크리스트 생성
- 호스트별 환경 문서 템플릿 생성
- AI quickstart stale 링크 검사

### 2.3 우선순위 3

- 코드 색인 초안 생성
- 병합 후 색인 재생성 보조
- PR 전 문서 동기화 리포트 생성

## 3. 자동화 후보 상세

### 3.1 백로그 템플릿 생성 보조

- 입력: 날짜, 작업 ID, 작업명
- 출력: 날짜별 백로그 신규 항목 초안

### 3.2 문서 메타데이터 누락 검사

- 입력: `docs/` 아래 Markdown 파일
- 출력: 누락된 메타데이터 필드 목록

### 3.3 허브/인덱스 링크 누락 검사

- 입력: 새로 추가되거나 이동된 문서 경로
- 출력: 갱신이 필요한 허브 문서 후보

### 3.4 변경 경로 기반 영향 문서 후보 출력

- 입력: `git diff --name-only`
- 출력: 변경 경로와 함께 볼 문서 후보 목록

## 4. 구현 준비 상태

현재 바로 착수 가능한 항목:

- 메타데이터 누락 검사 스크립트
- 허브/인덱스 링크 점검 스크립트
- 백로그 템플릿 생성 스크립트

추가 설계가 더 필요한 항목:

- 변경 경로 기반 영향 문서 후보 출력 규칙
- 색인 자동 생성 규칙
- 병합 후 재생성 워크플로우

## 5. 권장 구현 순서

1. 메타데이터 누락 검사
2. 백로그 템플릿 생성 보조
3. 허브/인덱스 링크 점검
4. 변경 경로 기반 영향 문서 후보 출력

## 6. 자동화 적용 원칙

- 자동화 결과는 사람이 검토 가능한 텍스트 출력이어야 한다.
- 문서 내용을 자동으로 확정하기보다 초안과 경고를 제공하는 방식을 우선한다.
- 저장소 규칙이 더 안정될 때까지는 CI 강제보다 로컬 보조를 우선한다.
