## 1. 작업 목적

현재 `CRS`, `SRS`, 역할 기반 UX, 프로토타입은 준비되어 있으나 이를 구현 가능한 구조로 연결하는 아키텍처 초안이 부재하다. 본 계획 문서는 아키텍처 초안 작성 전에 설계 범위와 산출물을 고정하고, 설계 진행 중 상태를 같은 문서 안에서 갱신하기 위한 기준으로 사용한다.

## 2. 초안 작성 범위

- Phase 1 통합 관리 플랫폼의 시스템 경계 정의
- 외부 연계 계층, 내부 표준화 계층, 조회/운영 계층의 분리 원칙 정리
- 과제 중심 데이터 모델의 상위 구조 정의
- 역할 기반 UI/운영 콘솔이 어떤 서비스 경계와 연결되는지 설명
- `AI` 보조 기능, 운영 도구, 감사 요구를 수용하는 구조 초안 정리

이번 단계에서는 구현 기술 확정보다 구조와 책임 경계를 우선 정의한다.

## 5. 설계 판단 기준

- 과제 중심 운영을 최상위 도메인 개념으로 유지한다.
- 외부 `SCM`, `CI/CD`, 정적분석 실행 계층은 대체보다 연계를 우선한다.
- 일반 업무 UX와 관리자 운영 UX는 서비스/권한 경계에서도 분리 가능해야 한다.
- 조직/인력 원천 반영과 내부 운영 마스터를 동시에 다룰 수 있어야 한다.
- `AI` 기능은 보조 기능이며, 감사 가능성과 사람 검토 흐름을 유지해야 한다.

## 7. 도메인 모델 구체화 전 확인 항목

도메인 모델을 엔터티 수준으로 구체화하기 전에 다음 항목을 먼저 확정하거나 최소한 가정으로 명시해야 한다.

### 7.1 업무 기준 식별자

- `work_item` 과 `project` 의 관계를 1:N 로 둘지, 일부 경우 N:N 연결을 허용할지
- `epic`, `story`, `task`, `sub-task` 를 타입과 계층 관계로 표현할지, 일부를 별도 엔터티로 둘지
- 관리 외 프로젝트가 관리 대상 전환 전까지 별도 식별자 체계를 가질지
- 외부 시스템 식별자와 내부 표준 식별자 중 어떤 값을 업무 기준 기본키로 삼을지
- `sprint`, `release`, 마일스톤, `WBS` 를 별도 계획 엔터티로 둘지
- `process_model` 을 고정 열거형으로 둘지, 확장 가능한 정의 모델로 둘지

### 7.2 상태 모델과 라이프사이클

- `work_item` 상태, 프로젝트 상태, 전환 상태, 마이그레이션 상태를 어디까지 독립 모델로 둘지
- 단순 조회용 상태와 승인/운영용 상태를 같은 상태 기계로 묶을지 분리할지
- `AI` 초안 상태와 본업무 상태를 어떤 경계로 분리할지
- `Agile`, `V-Model`, 혼합형 프로젝트에서 상태 모델을 공통 상태 + 프로세스별 상세 상태로 분리할지
- 신규 프로세스 모델이 들어올 때 상태/보드/계획 규칙을 설정만으로 추가할 수 있을지

### 7.3 조직/인력 기준정보 범위

- 조직장, 승인권자, 대행 승인자, 과제 책임자를 모두 별도 엔터티로 둘지 역할 관계로 표현할지
- 조직 개편 이력과 현재 조직 구조를 한 모델에서 표현할지 이력 모델을 분리할지
- 부재 일정이 개인 기준만 필요한지, 프로젝트/조직 단위 캘린더 엔터티가 별도로 필요한지

### 7.4 연계 데이터 보존 수준

- 원시 수집 데이터(`raw`)를 어느 수준까지 보존할지
- 표준화 후 스냅샷과 원천 참조를 둘 다 유지할지
- 조회 최적화를 위해 별도 읽기 모델이나 집계 스냅샷을 둘지

### 7.5 품질/개발 문맥의 경계

- 코드리뷰, 빌드, 테스트, 결함, 정적분석을 독립 도메인으로 더 쪼갤지
- 릴리스 준비도는 계산 뷰인지 별도 엔터티인지
- 품질 게이트 판정 기록을 이력으로 저장할지 계산 결과로만 둘지
- 보드, 간트, `WBS`, 릴리스 뷰를 저장 모델로 둘지 읽기 모델로 계산할지

### 7.6 권한/감사/예외 구조

- 권한을 역할 기반 `RBAC` 중심으로 둘지, 데이터 범위까지 포함한 정책 모델로 확장할지
- 예외 승인을 공통 모델 하나로 통합할지, 품질/권한/연계 예외를 분리할지
- 감사 로그를 업무 이벤트와 운영 이벤트로 분리 저장할지

### 7.7 `AI` 보조 모델의 저장 범위

- `AI` 입력 문맥을 전문 저장할지 해시/참조만 저장할지
- 규칙셋, 프롬프트 버전, 생성 모델 정보를 감사 범위에 포함할지
- `AI` 결과를 업무 엔터티의 부속물로 둘지 독립 엔터티로 둘지

## 8. 다음 업데이트 규칙
