## 1. 목적

본 문서는 통합 관리 플랫폼이 어떤 외부 시스템과 경계를 이루고 어떤 계층으로 데이터를 수집, 정규화, 전달하는지 정리한다. 실행 엔진을 대체하지 않고 연결과 운영 통제를 강화한다는 Phase 1 원칙을 구조적으로 설명하는 것이 목적이다.

## 2. 시스템 컨텍스트 초안

Phase 1 시스템은 크게 네 종류의 외부/내부 주체와 상호작용한다.

- 외부 원천/실행 시스템: `Jira`, `Bitbucket`, `Bamboo`, `Confluence`, 인사 `DB`, `Crowd`, 테스트 관리 도구, 정적분석/지표 도구
- 통합 관리 플랫폼: 표준화, 조회, 운영 통제, 역할 기반 UX를 제공하는 내부 시스템
- 사용자: 개발자, 프로젝트리더, 조직장, 시스템 관리자, `SE` 담당자, 테스트 담당자, 품질/감사 담당자
- `AI` 보조 계층: 코드리뷰, 단위테스트, `CI` 초안, 운영 도구 자동화를 보조하는 내부 또는 외부 `AI` 서비스

핵심 경계는 다음과 같다.

- 외부 시스템은 원본 데이터와 실제 실행 상태를 제공한다.
- 통합 관리 플랫폼은 원본을 대체하지 않고, 표준 모델로 연결하고 운영 문맥을 재구성한다.
- 조직/인력 변경은 `인사 DB -> 내부 마스터 -> 계정계` 순서의 통제된 흐름으로 처리한다.
- 관리자 콘솔은 연계 재실행, 권한 변경, 마이그레이션처럼 위험한 행위를 담당한다.

## 3. 논리 컴포넌트 초안

### 3.1 연계 수집 계층

- 외부 시스템별 커넥터
- 배치/이벤트/수동 동기화 실행기
- 수집 결과, 실패, 재처리 상태 관리자

이 계층은 외부 시스템과의 통신 책임을 갖고 내부 표준 모델에 전달할 원시 메타데이터를 수집한다.

### 3.2 표준화 및 기준정보 계층

- 표준 식별자 매핑 서비스
- 프로젝트/과제/조직/인력 기준정보 관리 서비스
- 조직 변경 영향 분석 및 마이그레이션 준비 서비스

이 계층은 외부 식별자와 내부 식별자의 연결, 원천 데이터와 내부 운영 마스터의 차이를 관리한다.

### 3.3 업무 도메인 연계 인터페이스

- 과제/프로젝트 추적 서비스
- 코드리뷰/빌드/산출물 문맥 서비스
- 품질/테스트/정적분석 문맥 서비스
- 공유 캘린더/가용성 서비스

이 계층은 표준화된 데이터를 응용 서비스가 사용할 수 있는 업무 문맥으로 변환한다.

### 3.4 운영 통제 연계 인터페이스

- 관리자 콘솔 서비스
- 권한/역할/감사 서비스
- 예외 승인 및 정책 관리 서비스

이 계층은 일반 사용자 흐름과 분리된 상태로 동작하며, 연계 재실행과 마이그레이션 같은 고위험 운영 행위를 통제한다.

### 3.5 백엔드 런타임 상세화 방향

시스템 통합 `DB` 구축용 백엔드 개발을 우선순위로 둘 경우, 위 논리 컴포넌트는 다음 런타임 컴포넌트로 더 세분된다.

- 연계 어댑터 계층: 외부 시스템별 커넥터, 증분 조회, 수집 커서 관리
- 동기화 오케스트레이션 계층: 스케줄러, 실행 상태 관리자, 수동 재실행 제어
- 원시 적재 계층: 원시 데이터 적재, 적재 오류 기록, 수집 메타데이터 관리
- 표준화/식별자 매핑 계층: 표준 속성 변환, 내부 기준키 연결, 참조 검증
- 도메인 쓰기 계층: `project`, `work_item`, 조직/인력 마스터 반영
- 운영 통제 계층: 정합성 오류 적재, 재처리 큐, 감사 로그

상세 구조와 유스케이스는 [integration_backend_component_draft.md](./integration_backend_component_draft.md)에서 별도로 관리한다.

## 4. 핵심 데이터 흐름 초안

### 4.1 과제 중심 통합 조회 흐름

1. 외부 시스템에서 요구사항, 변경, 빌드, 테스트, 품질 데이터를 수집한다.
2. 내부 표준화 계층이 공통 식별자와 과제 기준 관계를 재구성한다.
3. 업무 도메인 계층이 과제 워크스페이스, 프로젝트 운영, 품질 검증 화면용 뷰 모델을 구성한다.
4. 역할 기반 UI는 같은 과제 데이터를 역할별 필터와 우선순위에 맞게 노출한다.

### 4.2 조직 변경 반영 흐름

1. 인사 `DB` 변경을 수집한다.
2. 내부 마스터와 외부 계정계 간 차이를 계산한다.
3. 관리자 콘솔에서 영향 과제, 조직, 계정 범위를 검토한다.
4. 승인 후 내부 데이터와 계정 연계를 순차 반영한다.
5. 전 과정은 감사 로그로 남긴다.

### 4.3 `AI` 보조 흐름

1. 코드 변경, 테스트 현황, 품질 신호, 템플릿 정보 같은 입력을 수집한다.
2. `AI` 보조 계층이 초안을 생성한다.
3. 결과는 업무 화면 안에서 사람 검토 대상으로 노출된다.
4. 수용, 반려, 예외 승인, 재생성 상태는 운영 통제 계층에서 추적한다.

- 조직 변경 반영 시퀀스 다이어그램
